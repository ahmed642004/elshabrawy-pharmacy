- [ ] Cart-context auth-resync race — found while verifying reorder: cart-context.tsx's onAuthStateChange handler resyncs `items` from `mergeCartLists(server, local)` on EVERY auth event (not just SIGNED_IN), where `local` is read from localStorage. A background TOKEN_REFRESHED event firing right after a rapid burst of addItem() calls (e.g. multi-item reorder) can clobber items added in that window, since localStorage hasn't caught up yet. Rare in practice (needs a token refresh to coincide with rapid cart mutation) — done when: resync only runs on SIGNED_IN/SIGNED_OUT, or reads live React state instead of localStorage.
